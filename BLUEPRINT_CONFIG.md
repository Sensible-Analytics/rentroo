# Rentroo Render Blueprint Configuration

## 📋 Blueprint Overview

This document provides all configuration needed to deploy the Rentroo application using the Render Blueprint (`render.yaml`). The blueprint creates 7 backend services and defines their interconnections.

**Target Domain:** `api.rentroo.sensibleanalytics.co`  
**Frontend Domains:** `landlord.rentroo.sensibleanalytics.co`, `tenant.rentroo.sensibleanalytics.co`

## 🏗️ Services Summary

| Service | Purpose | Port | Health Check | Plan |
|---------|---------|------|--------------|------|
| **rentroo-gateway** | API gateway, routes requests | 8080 | `/health` | Free |
| **rentroo-api** | Core business logic | 8200 | `/health` | Free |
| **rentroo-authenticator** | JWT authentication | 8000 | `/health` | Free |
| **rentroo-tenantapi** | Tenant-specific APIs | 8250 | `/health` | Free |
| **rentroo-emailer** | Email notifications | 8400 | None | Free |
| **rentroo-pdfgenerator** | PDF document generation | 8300 | None | Free |
| **rentroo-resetservice** | Password reset service | 8900 | `/health` | Free |

## 🗄️ External Services Required

### 1. **MongoDB Atlas** (Database)
- **Plan:** M0 (Free tier, 512MB)
- **Region:** Choose closest to your users
- **Required by:** All services
- **Connection String:** `mongodb+srv://<user>:<password>@cluster.mongodb.net/rentroo`

### 2. **Redis** (Caching/Session)
- **Provider:** Upstash (free tier) or Render Redis
- **Required by:** Gateway, Authenticator
- **Connection URL:** `rediss://<user>:<password>@<host>.upstash.io:6379`

### 3. **Clerk** (Authentication)
- **Plan:** Free tier (10K MAU)
- **Required by:** Gateway, API (for future migration from JWT)
- **API Keys:** `CLERK_SECRET_KEY`, `CLERK_PUBLISHABLE_KEY`

### 4. **Resend** (Email)
- **Plan:** Free tier (3,000 emails/month)
- **Required by:** Emailer service
- **API Key:** `RESEND_API_KEY`
- **Sender Domain:** `noreply@rentroo.sensibleanalytics.co`

### 5. **Cloudflare R2** (File Storage)
- **Plan:** Free tier (10GB)
- **Required by:** PDFGenerator, API
- **Credentials:** `R2_ACCOUNT_ID`, `R2_ACCESS_KEY_ID`, `R2_SECRET_ACCESS_KEY`, `R2_BUCKET_NAME`

## 🔐 Environment Variables Configuration

### **Required in Render Dashboard**

You must configure these environment variables in the Render Dashboard for each service. Variables marked with `sync: false` in the blueprint must be manually set.

#### **All Services Common Variables**
| Variable | Value | Notes |
|----------|-------|-------|
| `MONGO_URL` | MongoDB Atlas connection string | Same for all services |
| `REDIS_URL` | Redis connection string | Gateway, Authenticator |
| `REDIS_PASSWORD` | Redis password | Gateway, Authenticator |
| `ACCESS_TOKEN_SECRET` | Random 32+ character string | Auto-generated by Render |
| `REFRESH_TOKEN_SECRET` | Random 32+ character string | Auto-generated by Render |
| `RESET_TOKEN_SECRET` | Random 32+ character string | Auto-generated by Render |
| `APPCREDZ_TOKEN_SECRET` | Random 32+ character string | Auto-generated by Render |
| `CIPHER_KEY` | Random 32+ character string | Auto-generated by Render |
| `CIPHER_IV_KEY` | Random 16+ character string | Auto-generated by Render |

#### **Service-Specific Variables**
| Service | Variable | Value |
|---------|----------|-------|
| **Gateway** | `ALLOWED_ORIGINS` | `https://landlord.rentroo.sensibleanalytics.co,https://tenant.rentroo.sensibleanalytics.co` |
| **API** | `RESTORE_DB` | `false` |
| **Authenticator** | `SIGNUP` | `true` |
| **Emailer** | `ALLOW_SENDING_EMAILS` | `true` |
| **Emailer** | `GMAIL_EMAIL` or `MAILGUN_API_KEY` or `SMTP_*` | Choose one email provider |
| **Emailer** | `EMAIL_FROM` | `noreply@rentroo.sensibleanalytics.co` |
| **PDFGenerator** | `UPLOAD_MAX_SIZE` | `2000000000` (~2GB) |

## 🌐 DNS Configuration

Add these DNS records at your domain registrar or Cloudflare:

| Type | Name | Value | Purpose |
|------|------|-------|---------|
| **CNAME** | `api` | `rentroo-gateway.onrender.com` | Backend API |
| **CNAME** | `*` (wildcard) | `rentroo-gateway.onrender.com` | Catch-all backend |
| **CNAME** | `landlord` | `cname.vercel-dns.com` | Landlord frontend |
| **CNAME** | `tenant` | `cname.vercel-dns.com` | Tenant frontend |

**Note:** SSL certificates are automatically provisioned by Render and Vercel.

## 🖥️ Vercel Frontend Configuration

### **Landlord Frontend**
- **Directory:** `webapps/landlord`
- **Build Command:** `cd ../.. && yarn build && cd webapps/landlord && yarn build`
- **Output Directory:** `.next`
- **Environment Variables:**
  - `NEXT_PUBLIC_GATEWAY_URL`: `https://api.rentroo.sensibleanalytics.co`
  - `NEXT_PUBLIC_BASE_PATH`: `/landlord`
  - `NPM_CLIENT`: `yarn` (critical - set in Vercel dashboard)

### **Tenant Frontend**
- **Directory:** `webapps/tenant`
- **Build Command:** `cd ../.. && yarn build && cd webapps/tenant && yarn build`
- **Output Directory:** `.next`
- **Environment Variables:**
  - `NEXT_PUBLIC_GATEWAY_URL`: `https://api.rentroo.sensibleanalytics.co`
  - `NEXT_PUBLIC_BASE_PATH`: `/tenant`
  - `NPM_CLIENT`: `yarn` (critical - set in Vercel dashboard)

## 🚀 Deployment Steps

### **Phase 1: External Services Setup**
1. **MongoDB Atlas:** Create M0 cluster, database user, whitelist IPs (0.0.0.0/0)
2. **Redis:** Create Upstash Redis instance
3. **Clerk:** Create application, add domains
4. **Resend:** Verify domain, create API key
5. **Cloudflare R2:** Create bucket, generate access keys

### **Phase 2: Render Deployment**
1. Go to Render Dashboard → **New** → **Blueprint**
2. Connect GitHub repository: `Sensible-Analytics/rentroo`
3. Select `render.yaml` from repository root
4. Configure environment variables as listed above
5. Click **Apply**

### **Phase 3: Vercel Deployment**
1. Connect GitHub repository to Vercel
2. Configure landlord and tenant projects as described above
3. Set `NPM_CLIENT=yarn` environment variable
4. Deploy both frontends

### **Phase 4: DNS & Verification**
1. Add DNS records as specified
2. Run health checks:
   ```bash
   curl https://api.rentroo.sensibleanalytics.co/health
   curl https://landlord.rentroo.sensibleanalytics.co
   curl https://tenant.rentroo.sensibleanalytics.co
   ```
3. Run E2E tests:
   ```bash
   ./scripts/run-e2e.sh https://landlord.rentroo.sensibleanalytics.co https://api.rentroo.sensibleanalytics.co
   ```

## ⚠️ Important Notes

1. **Free Tier Limitations:** Render free tier provides 750 hours/month total across all services. Services will sleep after 15 minutes of inactivity.
2. **Authentication Migration:** The current JWT authentication will need to be migrated to Clerk for production use.
3. **Email Provider:** Choose one email provider (Gmail, Mailgun, or SMTP) and configure accordingly.
4. **Backup Secrets:** Generate and securely store all secret keys before deployment.
5. **Database Backups:** Enable MongoDB Atlas backups for production data.

## 📞 Support

For issues with the deployment:
1. Check service logs in Render Dashboard
2. Verify all environment variables are correctly set
3. Ensure DNS records are properly configured
4. Run the validation script: `render blueprints validate`
